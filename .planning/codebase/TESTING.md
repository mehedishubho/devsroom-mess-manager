# Testing

Test framework, structure, patterns, and coverage.

## Framework

- **PHPUnit 12.5.30** (configured in `composer.json` and `phpunit.xml`)
- **Not Pest** (despite `pestphp/pest-plugin` being in Composer's `allow-plugins`)
- Test runner: `composer run test` or `php artisan test`

## Structure

```
tests/
├── TestCase.php              # Base class
├── Feature/
│   └── ExampleTest.php       # Asserts GET / returns 200
└── Unit/
    └── ExampleTest.php       # Empty stub
```

## Test Suites

PHPUnit defines two suites (`phpunit.xml`):

- **Unit**: `tests/Unit/` — isolated logic tests
- **Feature**: `tests/Feature/` — full HTTP/DB integration tests

## Test Environment

`phpunit.xml` overrides for testing:

| Setting | Value |
|---|---|
| `APP_ENV` | `testing` |
| `BCRYPT_ROUNDS` | `4` (faster hashing) |
| `BROADCAST_CONNECTION` | `null` |
| `CACHE_STORE` | `array` |
| `DB_CONNECTION` | `sqlite` |
| `DB_DATABASE` | `:memory:` |
| `MAIL_MAILER` | `array` (no real email) |
| `QUEUE_CONNECTION` | `sync` (runs inline) |
| `SESSION_DRIVER` | `array` |
| `PULSE_ENABLED` | `false` |
| `TELESCOPE_ENABLED` | `false` |
| `NIGHTWATCH_ENABLED` | `false` |

## Patterns

### Base TestCase

`Tests\TestCase` extends Laravel's base test class. Use `$this->get()`, `$this->post()`, etc. for HTTP tests, and Eloquent factories for data.

### RefreshDatabase

Not currently used in `ExampleTest`. For feature tests touching the DB, add `use RefreshDatabase;` to the test class.

### Factories

- `UserFactory.php` is the only factory
- Generated by Laravel skeleton; not customized yet

### Test Naming

- File: `{Subject}Test.php`
- Method: `test_` prefix with snake_case (e.g., `test_the_application_returns_a_successful_response`)

## Coverage

- **Current coverage**: ~0% — only the default `ExampleTest` exists
- No domain features tested yet (no domain features exist)
- No CI configured (no `.github/workflows` test job)

## Recommended Additions (When Implementing Features)

- **Feature tests** for every controller action (happy path + auth checks)
- **Form request tests** for validation rules
- **Policy tests** for authorization rules
- **Unit tests** for services / actions with business logic
- **Database tests** using `RefreshDatabase` trait
- **Browser tests** (Laravel Dusk) — optional, not currently set up

## Helpers Available

- `$this->actingAs($user)` — authenticate as a user
- `$this->assertDatabaseHas('table', [...])` — DB assertions
- `Event::fake()`, `Mail::fake()`, `Queue::fake()` — prevent side effects
- `$this->seed()` — run seeders in test

## Running Tests

```bash
# All tests
composer run test

# Single suite
php artisan test --testsuite=Unit
php artisan test --testsuite=Feature

# Single test
php artisan test --filter=test_the_application_returns_a_successful_response

# With coverage (requires pcov or xdebug)
php artisan test --coverage
```
