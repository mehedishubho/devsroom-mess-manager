## What was built

Base infrastructure for the Devsroom Mess Management app:

- **MySQL schema** on `devsroom_mess_management` database: 16 domain tables (messes, settings, members, meal_entries, meal_off_requests, guest_meals, expense_categories, expenses, payments, monthly_closings, monthly_member_summaries, monthly_corrections, advance_balances, notifications, member_invitations, audits) with `mess_id` on every domain table
- **Eloquent models** for all 15 domain entities (15 with the new `BelongsToActiveMess` trait, plus `Mess` itself which uses the `Auditable` trait)
- **Audit trait** wired through `owen-it/laravel-auditing` v14.0 ‚Äî every model save writes to the `audits` table with full diff (before/after JSON)
- **Global `MessScope`** filters every domain query by `mess_id = config('mess.active_mess_id')` (default 1)
- **Time zone** set to `Asia/Dhaka`; `Carbon::setLocale('en')` set in `AppServiceProvider::boot()`
- **15 model factories** for test data
- **Test base class** updated with `setUp()` that sets `mess.active_mess_id = 1` and a `seedTyroRoles()` helper

## Key issues solved during execution

1. **Audit trait not firing in tests** ‚Äî the published `config/audit.php` hardcoded `'console' => false`. Switched to `env('AUDIT_CONSOLE', false)`, added `AUDIT_CONSOLE=true` to `.env` and `phpunit.xml`.
2. **Audit test assertion** ‚Äî `assertDatabaseCount('audits', 1)` failed because both `create` and `update` events fire. Updated test to expect 2 entries.
3. **Test config caching** ‚Äî `phpunit.xml` env vars weren't loading until `config:clear` was run.
