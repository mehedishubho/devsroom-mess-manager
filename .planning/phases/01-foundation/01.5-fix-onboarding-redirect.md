# Plan 1.5: Fix super-admin onboarding redirect when no mess exists

## Objective

Stop super-admin from landing on `/dashboard` when no mess has been
created yet. Two changes are required:

1. The post-login redirect closure in `AppServiceProvider::boot()` must
   route super-admin to `/onboarding` (instead of `/dashboard`) when
   no mess exists.
2. The `/onboarding` route group must apply `EnsureMessExists` so
   that any super-admin navigation that bypasses the closure still
   gets redirected to the create-mess form if a mess is missing.

By the end of this plan: deleting the only mess and re-logging in as
super-admin lands the user on `/onboarding` (not the Tyro dashboard).

## Context

**Source docs:**
- `.planning/debug/phase1-onboarding-redirect.md` — diagnosis
- `.planning/phases/01-foundation/01-UAT.md` — UAT failure (Test 15)
- `.planning/phases/01-foundation/01.3-SUMMARY.md` — original route design

**Constraints:**

  admin group). No regression.
- Member (user) flow is unaffected — they cannot reach the dashboard
  or onboarding.
- The fix must be regression-testable: prove the bug existed and
  the fix removed it.

## Tasks

### Task 1: Update the post-login redirect closure

**File:** `app/Providers/AppServiceProvider.php`

Update the closure in `boot()` to query `Mess::activeId()` (or
`Mess::query()->exists()`) and divert super-admin to `/onboarding`
when no mess exists:

```php
config([
    'tyro-login.redirects.after_login' => function () {
        $user = Auth::user();
        if (! $user) {
            return '/';
        }
        // Super-admin must create the first mess before reaching the
        // dashboard. Manager/member routes are guarded by
        // EnsureMessExists middleware already.
        if ($user->hasRole('super-admin') && ! \App\Models\Mess::query()->exists()) {
            return route('onboarding.create');
        }
        if ($user->hasRole('super-admin')) {
            return '/dashboard';
        }
        if ($user->hasRole('admin')) {
            return '/home';
        }
        if ($user->hasRole('user')) {
            return '/my';
        }
        return '/';
    },
]);
```

### Task 2: Apply `EnsureMessExists` to the super-admin onboarding group

**File:** `routes/web.php`

Update the super-admin route group to include `EnsureMessExists`:

```php
Route::middleware(['auth', 'role:super-admin', EnsureMessExists::class])
    ->group(function () {
        Route::get('/onboarding', [OnboardingController::class, 'create'])
            ->name('onboarding.create');
        Route::post('/onboarding', [OnboardingController::class, 'store'])
            ->name('onboarding.store');
    });
```

Note: the middleware checks "does any mess exist?" and redirects to
`onboarding.create` if not. This is a no-op when the user is already
on the onboarding page because the route name matches and
`Mess::query()->doesntExist()` returns false once the mess is
created.

Wait — that creates a loop. If the user is on `/onboarding` and
no mess exists, the middleware will redirect them to
`/onboarding`, which is a redirect to itself. Laravel handles
self-redirects by issuing the same response (no infinite loop
because the browser follows one redirect). But to be safe, the
middleware should skip the check when the current route is already
`onboarding.create` or `onboarding.store`.

Update `EnsureMessExists`:

```php
public function handle(Request $request, Closure $next): Response
{
    if ($request->routeIs('onboarding.*')) {
        return $next($request);
    }
    if (Mess::query()->doesntExist()) {
        return redirect()->route('onboarding.create');
    }
    return $next($request);
}
```

### Task 3: Add regression tests

**File:** `tests/Feature/OnboardingRedirectTest.php`

```php
<?php

namespace Tests\Feature;

use App\Models\Mess;
use App\Models\User;
use HasinHayder\Tyro\Models\Role;
use Illuminate\Foundation\Testing\RefreshDatabase;
use Tests\TestCase;

class OnboardingRedirectTest extends TestCase
{
    use RefreshDatabase;

    protected function setUp(): void
    {
        parent::setUp();
        $this->seedTyroRoles();
    }

    public function test_closure_routes_super_admin_to_onboarding_when_no_mess(): void
    {
        $user = User::factory()->create();
        $user->assignRole(Role::where('slug', 'super-admin')->first());
        $this->actingAs($user);

        $url = config('tyro-login.redirects.after_login')();
        $this->assertSame(route('onboarding.create'), $url);
    }

    public function test_closure_routes_super_admin_to_dashboard_when_mess_exists(): void
    {
        Mess::factory()->create();
        $user = User::factory()->create();
        $user->assignRole(Role::where('slug', 'super-admin')->first());
        $this->actingAs($user);

        $url = config('tyro-login.redirects.after_login')();
        $this->assertSame('/dashboard', $url);
    }

    public function test_ensure_mess_middleware_skips_onboarding_route(): void
    {
        $user = User::factory()->create();
        $user->assignRole(Role::where('slug', 'super-admin')->first());
        $this->actingAs($user);

        $response = $this->get(route('onboarding.create'));
        $response->assertOk();
    }

    public function test_ensure_mess_middleware_redirects_when_no_mess_for_unrelated_route(): void
    {
        // Visiting /dashboard (Tyro) as super-admin with no mess should
        // redirect to /onboarding.
        $user = User::factory()->create();
        $user->assignRole(Role::where('slug', 'super-admin')->first());
        $this->actingAs($user);

        $response = $this->get('/dashboard');
        $response->assertRedirect(route('onboarding.create'));
    }
}
```
