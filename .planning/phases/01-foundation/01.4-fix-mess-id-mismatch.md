# Plan 1.4: Fix mess_id mismatch

## Objective

Stop reading `config('mess.active_mess_id')` directly as the integer that
goes into `mess_id` columns. Instead, resolve the active mess id at runtime
from the `messes` table, with `config('mess.active_mess_id')` as an
optional env override (used only in tests / multi-mess installs).

By the end of this plan: inviting a member works regardless of the actual
auto-increment id of the mess row; a regression test proves the fix.

## Context

**Source docs:**
- `.planning/debug/phase1-mess-id-mismatch.md` — diagnosis
- `.planning/phases/01-foundation/01-UAT.md` — UAT failure (Test 10)
- `.planning/phases/01-foundation/01.3-SUMMARY.md` — original MemberInviteController design

**Constraints:**
- No data loss. Existing dev data (mess at id=1 after the manual re-anchor)
  must continue to work.
- Backward compatible: the `mess.active_mess_id` env value still works for
  tests that pre-set it.
- No change to the public API: controllers still return the same responses.

## Tasks

### Task 1: Add `Mess::activeId(): ?int` static helper

**File:** `app/Models/Mess.php`

Add a static method that returns the first mess id in the database (or the
configured override if set), with a per-request static cache so we don't
query on every call.

```php
private static ?int $activeIdCache = null;

public static function activeId(): ?int
{
    if (self::$activeIdCache !== null) {
        return self::$activeIdCache;
    }

    $override = config('mess.active_mess_id');
    if (is_int($override) || (is_string($override) && ctype_digit($override))) {
        // Honour the override only if a mess actually exists at that id.
        $id = (int) $override;
        if (Mess::query()->whereKey($id)->exists()) {
            return self::$activeIdCache = $id;
        }
    }

    $id = Mess::query()->orderBy('id')->value('id');
    return self::$activeIdCache = $id ? (int) $id : null;
}

public static function forgetActiveIdCache(): void
{
    self::$activeIdCache = null;
}
```

### Task 2: Update `MessScope` to use the helper

**File:** `app/Models/Scopes/MessScope.php`

Replace `config('mess.active_mess_id')` with `Mess::activeId()`. The scope
runs on every model query, so the static cache is essential.

```php
public function apply(Builder $builder, Model $model): void
{
    $messId = Mess::activeId();
    if ($messId !== null) {
        $builder->where($model->getTable().'.mess_id', $messId);
    }
}
```

### Task 5: Add a regression test

**File:** `tests/Feature/Mess/InviteMessIdMismatchTest.php`

```php
<?php

namespace Tests\Feature\Mess;

use App\Mail\SetPasswordMail;
use App\Models\Mess;
use App\Models\User;
use HasinHayder\Tyro\Models\Role;
use Illuminate\Foundation\Testing\RefreshDatabase;
use Illuminate\Support\Facades\Mail;
use Tests\TestCase;

class InviteMessIdMismatchTest extends TestCase
{
    use RefreshDatabase;

    protected function setUp(): void
    {
        parent::setUp();
        $this->seedTyroRoles();
    }

    public function test_invite_works_when_mess_is_at_a_non_default_id(): void
    {
        // Simulate the bug: env says id=1, but the actual mess is at id=42.
        config(['mess.active_mess_id' => 1]);
        $mess = Mess::factory()->create(['id' => 42, 'name' => 'Real Mess']);
        // Sanity: the env-supplied id is wrong but a mess exists.
        $this->assertFalse(Mess::query()->whereKey(1)->exists());
        $this->assertTrue(Mess::query()->whereKey(42)->exists());

        $admin = User::factory()->create();
        $admin->assignRole(Role::where('slug', 'admin')->first());

        Mail::fake();
        $this->actingAs($admin);

        $controller = app(\App\Http\Controllers\Mess\MemberInviteController::class);
        $method = (new \ReflectionClass($controller))->getMethod('store');
        $method->setAccessible(true);

        $request = \App\Http\Requests\Mess\InviteMemberRequest::create(
            route('mess.members.invite.store'), 'POST', ['email' => 'new@test.com']
        );
        $request->setContainer(app());
        $request->setRedirector(app('redirect'));
        $request->setUserResolver(fn () => $admin);
        $request->validateResolved();

        // Before the fix: throws QueryException (FK violation).
        // After the fix: returns a redirect.
        $response = $method->invoke($controller, $request);
        $this->assertInstanceOf(\Illuminate\Http\RedirectResponse::class, $response);

        $this->assertDatabaseHas('member_invitations', [
            'email' => 'new@test.com',
            'mess_id' => 42, // the real mess id, not the env value
        ]);
    }

    public function test_active_id_helper_returns_first_mess(): void
    {
        $this->assertNull(Mess::activeId());

        $mess = Mess::factory()->create();
        Mess::forgetActiveIdCache();
        $this->assertSame((int) $mess->id, Mess::activeId());
    }

    public function test_active_id_helper_falls_back_to_override_when_mess_exists(): void
    {
        config(['mess.active_mess_id' => 99]);
        $mess = Mess::factory()->create(['id' => 99]);
        Mess::forgetActiveIdCache();
        $this->assertSame(99, Mess::activeId());
    }
}
```
