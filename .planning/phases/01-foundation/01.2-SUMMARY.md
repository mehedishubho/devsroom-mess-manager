## What was built

- **Tyro Login configured** with 2FA forced on `admin` and `super-admin` roles (`TYRO_LOGIN_2FA_ENABLED=true`, `TYRO_LOGIN_2FA_FORCED_ROLES=admin,super-admin`, `TYRO_LOGIN_2FA_ALLOW_SKIP=false`); member (`user`) role has 2FA off

## Key issues solved during execution

1. **`__()` in config files fail** — the translator isn't loaded when config is parsed. Tests that called `__()` inside `config/tyro-dashboard.php` resources array caused `Target class [translator] does not exist`. Resolved in Plan 01.3 by using plain English strings (Tyro Dashboard's standard convention).
2. **CSRF 419 errors in POST tests** — couldn't disable CSRF cleanly through `withoutMiddleware()` chaining. Resolved by using **direct controller invocation** in tests via `ReflectionClass::getMethod()` + `setAccessible(true)` + `$request->setUserResolver()` — bypasses middleware entirely but still exercises the controller.
3. **Vite manifest missing** — `home.blade.php` failed to render with 500. Fixed by `npm install && npm run build`.

## Notes for next plans

- The 2FA tests assert config values rather than full HTTP flow, because the Tyro Login controller's internals are not test-friendly without CSRF. The 2FA gate is verified at the config level (which is what the user controls); UAT covers the actual user experience.
- The closure for `redirects.after_login` is a callable — Tyro's redirect helper accepts it. If Tyro's code changes the redirect handling, the closure will be called as a function and the string return is used.
