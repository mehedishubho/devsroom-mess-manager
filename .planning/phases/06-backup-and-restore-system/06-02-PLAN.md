**1. `app/Support/BackupPathResolver.php`** — a small pure-PHP class:
```php
namespace App\Support;

use Illuminate\Filesystem\Filesystem;
use Illuminate\Support\Str;

class BackupPathResolver
{
    public function __construct(private readonly Filesystem $files) {}

    /**
     * Locate the SQL dump inside an extracted spatie backup tree.
     * Pitfall 1 (spatie/laravel-backup Issue #1389): the db-dumps/ folder may be
     * nested under the source base path. Use a recursive glob, not a fixed path.
     *
     * @throws \RuntimeException when zero or more than one .sql found
     */
    public function locateSqlDump(string $extractedRoot): string
    {
        $matches = $this->files->glob($extractedRoot.'/**/db-dumps/*.sql');
        if (count($matches) === 1) {
            return $matches[0];
        }
        // Fallback: legacy flat layout
        $matches = $this->files->glob($extractedRoot.'/db-dumps/*.sql');
        if (count($matches) === 1) {
            return $matches[0];
        }
        throw new \RuntimeException(
            count($matches) === 0
                ? 'No SQL dump found under db-dumps/ in the extracted backup.'
                : 'Multiple SQL dumps found in the backup; cannot auto-select.'
        );
    }
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Backup zip (object storage) → mysql live DB | The restore crosses this boundary INTO the live DB. Highest-risk surface in the entire app. |
| Scheduler/cron → artisan commands | Scheduled commands run with the web user's privileges; spatie + RestoreTestRun are cron-invoked, not user-invoked. |
| spatie internal failure events → NotificationService | spatie emits events on failure; our listener turns them into in-app notifications. |
| CloseMonthJob completion → backup:run | The post-close hook fires backup:run; a backup failure must NEVER break the close path. |
| Restore-test scratch DB | Separate DB on the same host; wiped every run; never exposed to the web. |
