# Plan 2.5: Bazar + Fixed Expenses + Categories — Summary

## What was built

Manager's expense entry surfaces (bazar + fixed), expense category manager with default lock, schema reconciliation (kind moved from expenses.expense_type to expense_categories.kind), default categories seeded.

## Key changes

- **Migrations** — added `kind` enum to `expense_categories`, dropped `expense_type` from `expenses` (D-20)
- **ExpenseCategory model** — added `Auditable` trait, `kind` fillable, `expenses()` relation
- **Expense model** — added `Auditable` trait, removed `expense_type` fillable
- **ExpenseKind** — new support class (bazar/fixed/other)
- **ExpenseCategorySeeder** — 13 default categories (7 bazar, 6 fixed), idempotent via firstOrCreate
- **DatabaseSeeder** — registers ExpenseCategorySeeder
- **Services** — ExpenseService (list/create with receipt upload), ExpenseCategoryService (list/create/delete with default lock)
- **Controllers** — ExpenseController (index + bazar create/store + fixed create/store), ExpenseCategoryController (index + store + destroy)
- **Form Requests** — StoreExpenseRequest (kind-aware category rule), StoreCategoryRequest
- **Views** — expenses/index, expenses/bazar/create, expenses/fixed/create, categories/index (with form)
- **Sidebar** — added Expenses + Categories links
- **Routes** — 7 new routes for expenses + categories

## Tests added

- `ExpenseTest` (3 tests) — view index, create bazar, create fixed
- `ExpenseCategoryTest` (4 tests) — view categories, defaults seeded, default can't be deleted, custom can be deleted
- `ExpenseAuditTest` (1 test) — creating expense writes audit log
