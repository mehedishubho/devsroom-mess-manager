# Plan 2.2: Member Self-View + Meal Off Request — Summary

## What was built

Member's self-service surface at `/my`: tabbed profile/meal-off/meals interface, meal off request form with status list, photo upload, emergency contact editing. Manager on-behalf meal off submission from member profile.

## Key changes

- **MealOffRequest model** — added `Auditable` trait (AUDIT-01)
- **MealOffStatus** — new support class (pending/approved/rejected)
- **User model** — added `member()` HasOne relation + `getMemberOrNull()` helper
- **MyController** — replaced placeholder with full tabbed interface (index, updateProfile, storeMealOff)
- **ManagerMealOffController** — manager on-behalf submission
- **Form Requests** — UpdateMyProfileRequest (D-24 limited fields), StoreMealOffRequest (member self), StoreManagerMealOffRequest (admin)
- **TabNav component** — accessible tab strip (role="tablist", aria-selected, aria-current)
- **Views** — my.blade.php (tabbed), _profile, _meal-off, _meals partials, no-member placeholder
- **Member show view** — added manager on-behalf meal off form section
- **Routes** — `my.profile.update`, `my.meal-off.store`, `mess.members.meal-off.store`
- **MealOffRequestFactory** — added pending/approved/rejected states

## Tests added

- `MyMealOffTest` (3 tests) — submit meal off, validation, audit log writes
