# Plan 3.3: Live bill preview + 1-hour cache

## Objective

Build the "if we closed today" bill preview (PREVIEW-01 to PREVIEW-05) as a cached read-only computation: meal rate = `total_bazar / total_meals` (CLOSE-04), fixed cost share prorated by days (CLOSE-05 / D-13), per-member bill = `meal_cost + fixed_share - advance_applied - payments_received`. Cache the result for 1 hour (D-14) under a single key per `(mess, year, month)`. Expose manager full-preview at `/mess/bill-preview` (or `?year=YYYY&month=M`), a manager dashboard widget at `/home`, and a member widget at `/my` (D-16). Wire cache invalidation hooks (D-15) on all relevant write events from the `PaymentService` and the existing meal/expense/guest-meal/meal-off services.

By the end of this plan: the manager sees total meals, total bazar, computed meal rate, and a per-member breakdown for the current month; the member sees only their own row; writes to meals, expenses, guest meals, meal-off requests, and payments invalidate the cache for that `(year, month)`. The close logic itself is **not** in this plan — it ships in Plan 3.4. The preview is a **read** of the same data, never a write.

**Requirements covered:** PREVIEW-01, PREVIEW-02, PREVIEW-03, PREVIEW-04, PREVIEW-05, CLOSE-04, CLOSE-05, CLOSE-06 (preview-only — full close flow is Plan 3.4).

**Constraints:**
- Meal rate = `total_bazar / total_meals` — denominator is sum of `meal_value` for active meals of members active for the whole month (D-12). A member who joined day 15 still has their meals counted **if** they were active for the whole month, but members with `joining_date > month_start` or `leaving_date < month_end` are **excluded from the denominator** when computing the average.
- Fixed cost share is prorated by days: `total_fixed * (days_member_was_active_this_month / total_days_in_month)` (D-13).
- Cache key: `bill-preview:{mess_id}:{year}-{month}` (D-14). Cached 1 hour (PREVIEW-05). `Cache::remember()` is fine.
- For the **current** month, preview uses data up to today. For a **past** month with no close, preview uses data up to the last day of that month. For a **past** month **with** a close, preview reads `MonthlyMemberSummary` rows (D-16) — but `MonthlyClosing` is built in Plan 3.4, so this plan only handles "no close" cases (past month with no close is the same as current month; just with explicit end date).
- Guest meal charges are **already locked at entry** per Phase 2 D-17; do not recompute at preview time (D-17).
- Cache invalidation: a write to `meal_entries`, `meal_off_requests`, `guest_meals`, `expenses`, or `payments` for that `(mess, year, month)` calls `BillPreviewInvalidator::for($year, $month)`. Use Laravel model `saved` / `deleted` events (D-15) — register listeners in `AppServiceProvider::boot()`.
- Use `Mess::activeId()` everywhere for `mess_id` writes/reads.
- This plan reads only — no writes to `monthly_closings`, `monthly_member_summaries`, or `advance_balances`.

### Task 1: Add `BillPreviewService` (read-only computation, cached)

**File:** `app/Services/BillPreviewService.php` (new)

```php
<?php

namespace App\Services;

use App\Models\Expense;
use App\Models\GuestMeal;
use App\Models\MealEntry;
use App\Models\MealOffRequest;
use App\Models\Member;
use App\Models\Mess;
use App\Models\Payment;
use App\Support\ExpenseKind;
use App\Support\MealOffStatus;
use App\Support\MemberStatus;
use App\Support\PaymentType;
use Carbon\Carbon;
use Illuminate\Support\Collection;
use Illuminate\Support\Facades\Cache;
use Illuminate\Support\Facades\DB;

class BillPreviewService
{
    public const CACHE_TTL_SECONDS = 3600; // 1 hour (PREVIEW-05)

    /**
     * Compute (or fetch cached) preview for a (year, month).
     * @return array<string, mixed>
     */
    public function preview(int $year, int $month, ?Carbon $asOf = null): array
    {
        $messId = Mess::activeId();
        $asOf = $asOf ?: Carbon::create($year, $month, 1)->endOfMonth();

        $cacheKey = "bill-preview:{$messId}:{$year}-{$month}";

        return Cache::remember($cacheKey, self::CACHE_TTL_SECONDS, function () use ($messId, $year, $month, $asOf) {
            return $this->compute($messId, $year, $month, $asOf);
        });
    }

    /**
     * Invalidate the cache for a (year, month). Safe to call from anywhere.
     */
    public function invalidate(int $year, int $month): void
    {
        $messId = Mess::activeId();
        Cache::forget("bill-preview:{$messId}:{$year}-{$month}");
    }

    /**
     * @return array<string, mixed>
     */
    private function compute(int $messId, int $year, int $month, Carbon $asOf): array
    {
        $monthStart = Carbon::create($year, $month, 1)->startOfDay();
        $monthEnd = $monthStart->copy()->endOfMonth()->endOfDay();
        $daysInMonth = $monthStart->daysInMonth;
        $effectiveEnd = $asOf->lt($monthEnd) ? $asOf : $monthEnd;

        // Active members for the month (D-12): joining_date <= month_start AND (leaving_date IS NULL OR leaving_date >= month_end)
        $members = Member::query()
            ->where('status', MemberStatus::ACTIVE)
            ->whereDate('joining_date', '<=', $monthStart->toDateString())
            ->where(function ($q) use ($monthEnd) {
                $q->whereNull('leaving_date')->orWhereDate('leaving_date', '>=', $monthEnd->toDateString());
            })
            ->orderBy('name')
            ->get();

        $memberIds = $members->pluck('id');

        // Total bazar: expenses of kind=bazar with date in [month_start, effectiveEnd]
        $totalBazar = (float) Expense::query()
            ->whereHas('category', fn ($q) => $q->where('kind', ExpenseKind::BAZAR))
            ->whereBetween('date', [$monthStart->toDateString(), $effectiveEnd->toDateString()])
            ->sum('amount');

        // Total fixed: expenses of kind=fixed
        $totalFixed = (float) Expense::query()
            ->whereHas('category', fn ($q) => $q->where('kind', ExpenseKind::FIXED))
            ->whereBetween('date', [$monthStart->toDateString(), $effectiveEnd->toDateString()])
            ->sum('amount');

        // Meal entries for active members in window
        $entries = MealEntry::query()
            ->whereIn('member_id', $memberIds)
            ->whereBetween('date', [$monthStart->toDateString(), $effectiveEnd->toDateString()])
            ->get();

        // Approved meal-off windows in window
        $mealOffs = MealOffRequest::query()
            ->whereIn('member_id', $memberIds)
            ->where('status', MealOffStatus::APPROVED)
            ->where(function ($q) use ($monthStart, $effectiveEnd) {
                $q->whereDate('from_date', '<=', $effectiveEnd->toDateString())
                  ->whereDate('to_date', '>=', $monthStart->toDateString());
            })
            ->get();

        // Guest meals for these members in window
        $guestMeals = GuestMeal::query()
            ->whereIn('member_id', $memberIds)
            ->whereBetween('date', [$monthStart->toDateString(), $effectiveEnd->toDateString()])
            ->get();

        // Payments for these members in window
        $payments = Payment::query()
            ->whereIn('member_id', $memberIds)
            ->whereBetween('date', [$monthStart->toDateString(), $effectiveEnd->toDateString()])
            ->get();

        // Compute per-member
        $rows = $members->map(function (Member $member) use ($entries, $mealOffs, $guestMeals, $payments, $monthStart, $monthEnd, $daysInMonth) {
            $memberEntries = $entries->where('member_id', $member->id);

            // Per-meal counts: meal_value*count
            $mealsCount = 0.0;
            foreach ($memberEntries as $e) {
                if ($e->breakfast) {
                    $mealsCount += 0.5;
                }
                if ($e->lunch) {
                    $mealsCount += 1.0;
                }
                if ($e->dinner) {
                    $mealsCount += 1.0;
                }
            }

            // Subtract meal-off days: each approved meal-off day removes one full day's worth of meals
            // (1 day ≈ 2.5 meal_value: 0.5 + 1 + 1) — matches the existing MealOffStatus implementation in Phase 2
            $offDays = 0;
            foreach ($mealOffs->where('member_id', $member->id) as $off) {
                $offStart = $off->from_date->lt($monthStart) ? $monthStart : $off->from_date;
                $offEnd = $off->to_date->gt($monthEnd) ? $monthEnd : $off->to_date;
                $offDays += $offStart->diffInDays($offEnd) + 1;
            }
            $mealsCount = max(0.0, $mealsCount - ($offDays * 2.5));

            // Guest meal charges (locked at entry per D-17)
            $guestCharge = (float) $guestMeals->where('member_id', $member->id)->sum('charge_amount');

            // Fixed cost share prorated by days (D-13). If member is active for full month, full share.
            $activeDays = $daysInMonth;
            $fixedShare = (float) ($this->totalFixedGlobal * ($activeDays / $daysInMonth) / max(1, $this->activeMemberCountGlobal));

            $mealCost = 0.0; // filled in by caller after meal rate is computed
            $grossBill = 0.0;
            $paymentsReceived = (float) $payments->where('member_id', $member->id)->where('type', PaymentType::BILL_PAYMENT)->sum('amount');
            $advanceBalance = (float) optional($member->advanceBalance)->balance ?? 0.0;
            $advanceApplied = min($advanceBalance, max(0.0, $grossBill - $paymentsReceived));
            $netBill = $grossBill - $advanceApplied - $paymentsReceived;

            return (object) [
                'member' => $member,
                'total_meals' => $mealsCount,
                'meal_cost' => $mealCost,
                'fixed_cost_share' => $fixedShare,
                'guest_meal_charge' => $guestCharge,
                'gross_bill' => $grossBill,
                'advance_applied' => $advanceApplied,
                'payments_received' => $paymentsReceived,
                'net_bill' => $netBill,
            ];
        });

        $totalMeals = (float) $rows->sum('total_meals');
        $mealRate = $totalMeals > 0 ? round($totalBazar / $totalMeals, 4) : 0.0;

        // Second pass: meal cost + gross bill with computed rate
        $this->totalFixedGlobal = $totalFixed;
        $this->activeMemberCountGlobal = max(1, $members->count());

        $rows = $rows->map(function ($row) use ($mealRate, $totalFixed, $members, $daysInMonth) {
            $row->meal_cost = round((float) $row->total_meals * $mealRate, 2);
            $row->gross_bill = round((float) $row->meal_cost + (float) $row->fixed_cost_share + (float) $row->guest_meal_charge, 2);
            $row->advance_applied = min((float) optional($row->member->advanceBalance)->balance ?? 0.0, max(0.0, $row->gross_bill - (float) $row->payments_received));
            $row->net_bill = round((float) $row->gross_bill - (float) $row->advance_applied - (float) $row->payments_received, 2);

            return $row;
        });

        return [
            'year' => $year,
            'month' => $month,
            'as_of' => $effectiveEnd->toDateString(),
            'total_bazar' => round($totalBazar, 2),
            'total_fixed' => round($totalFixed, 2),
            'total_meals' => round($totalMeals, 2),
            'meal_rate' => $mealRate,
            'member_count' => $members->count(),
            'rows' => $rows,
        ];
    }

    private float $totalFixedGlobal = 0.0;
    private int $activeMemberCountGlobal = 1;
}
```

**Note:** The `totalFixedGlobal` / `activeMemberCountGlobal` instance state is a smell — refactor to pass the rate computation into the per-member map directly. A cleaner approach: compute `mealRate` and `fixedShare` once, then iterate members. The plan keeps the two-pass structure because it computes member-level `total_meals` (which depends on entries + meal-offs) before the rate. The simpler form:

```php
// Inside compute():
$fixedPerActiveDay = $totalFixed / $daysInMonth;
// per member fixed share = fixedPerActiveDay * (days_member_was_active) / 1
// equal share per member for full-month: fixedPerActiveDay * daysInMonth / $members->count()
```

**Replace the per-member closure with this cleaner version:**

```php
$fixedPerActiveDay = $totalFixed / max(1, $daysInMonth);
$perMemberFullFixedShare = $fixedPerActiveDay * $daysInMonth / max(1, $members->count());

$rows = $members->map(function (Member $member) use ($entries, $mealOffs, $guestMeals, $payments, $monthStart, $monthEnd, $daysInMonth, $perMemberFullFixedShare) {
    // ... same as before but use $perMemberFullFixedShare instead of $fixedShare
    $fixedShare = $perMemberFullFixedShare;
    // ...
});

// Then in second pass:
$rows = $rows->map(function ($row) use ($mealRate) {
    $row->meal_cost = round((float) $row->total_meals * $mealRate, 2);
    $row->gross_bill = round($row->meal_cost + $row->fixed_cost_share + $row->guest_meal_charge, 2);
    $row->advance_applied = min((float) optional($row->member->advanceBalance)->balance ?? 0.0, max(0.0, $row->gross_bill - $row->payments_received));
    $row->net_bill = round($row->gross_bill - $row->advance_applied - $row->payments_received, 2);
    return $row;
});
```

### Task 2: Add `BillPreviewInvalidator` (small helper called from listeners)

**File:** `app/Services/BillPreviewInvalidator.php` (new)

```php
<?php

namespace App\Services;

use Carbon\Carbon;
use Illuminate\Database\Eloquent\Model;

class BillPreviewInvalidator
{
    public function __construct(private readonly BillPreviewService $service) {}

    /**
     * Invalidate cache for the (year, month) of the model's date column.
     * $dateColumn is the column name on the model that holds the event date.
     */
    public function forModelDate(Model $model, string $dateColumn = 'date'): void
    {
        $date = $model->{$dateColumn} ?? null;
        if (! $date) {
            return;
        }

        $carbon = $date instanceof \DateTimeInterface ? Carbon::instanceOf($date) : Carbon::parse($date);
        $this->service->invalidate((int) $carbon->year, (int) $carbon->month);
    }
}
```

<?php

namespace App\Http\Controllers\Mess;

use App\Http\Controllers\Controller;
use App\Models\MonthlyClosing;
use App\Services\BillPreviewService;
use Carbon\Carbon;
use Illuminate\Http\Request;
use Illuminate\View\View;

class BillPreviewController extends Controller
{
    public function __construct(private readonly BillPreviewService $service) {}

    public function index(Request $request): View
    {
        $now = Carbon::now();
        $year = (int) $request->query('year', $now->year);
        $month = (int) $request->query('month', $now->month);

        $preview = $this->service->preview($year, $month);

        $closed = MonthlyClosing::query()
            ->where('year', $year)
            ->where('month', $month)
            ->exists();

        return view('mess.bill-preview.index', compact('preview', 'year', 'month', 'closed'));
    }
}

<?php

namespace App\Http\Controllers\My;

use App\Http\Controllers\Controller;
use App\Services\BillPreviewService;
use Carbon\Carbon;
use Illuminate\Http\Request;
use Illuminate\View\View;

class MyBillPreviewController extends Controller
{
    public function __construct(private readonly BillPreviewService $service) {}

    public function index(Request $request): View
    {
        $member = $request->user()->getMemberOrNull();
        if (! $member) {
            return view('my.no-member');
        }

        $now = Carbon::now();
        $year = (int) $request->query('year', $now->year);
        $month = (int) $request->query('month', $now->month);

        $preview = $this->service->preview($year, $month);
        $myRow = $preview['rows']->firstWhere('member.id', $member->id);

        return view('my._bill-preview', [
            'preview' => $preview,
            'myRow' => $myRow,
            'year' => $year,
            'month' => $month,
        ]);
    }
}
