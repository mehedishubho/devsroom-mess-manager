# Plan 3.2: Advance balance + due balance + carry-forward

## What was built

- Migration `2026_06_18_000000_add_due_balance_to_advance_balances` adds the `due_balance` column (decimal 10,2, default 0)
- `AdvanceBalance` model: `due_balance` added to `#[Fillable]` and casts, NOT `Auditable` per D-11
- `Member::advanceBalance()` `hasOne` relation
- `App\Services\AdvanceBalanceService` with three methods:
  - `applyPayment(Payment)` — only `advance_deposit` triggers an `advance_balance` increase (D-07)
  - `adjust(memberId, amount, reason, enteredBy)` — manual manager adjustment (positive = credit, negative = due), rejects 0 amount
  - `carryForward(memberId, amount)` — for `MonthCloseService` to call in Plan 3.4
- `PaymentService::create()` now calls `AdvanceBalanceService::applyPayment()` after writing the payment
- `App\Http\Controllers\Mess\AdvanceBalanceController` — `index` (list active members with balances), `adjust` (form), `storeAdjust`
- `App\Http\Requests\Mess\AdjustAdvanceBalanceRequest` — admin auth, `amount` rule with `not_in:0`, required `reason`
- `AdvanceBalanceFactory` — `withAdvance()` + `withDue()` state methods
- Manager views: `index.blade.php`, `_list.blade.php` (table + mobile cards), `adjust.blade.php`
- Member views: `my._advance-balance.blade.php` partial + new "Advance balance" tab in `my.blade.php`
- Sidebar nav link + home dashboard card
