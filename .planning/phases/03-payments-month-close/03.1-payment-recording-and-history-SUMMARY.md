# Plan 3.1: Payment recording + payment history

## What was built

Manager-side payment recording (single form with segmented `Bill payment` / `Advance deposit` control) and member-side read-only payment history.

- `App\Support\PaymentType` — `BILL_PAYMENT` / `ADVANCE_DEPOSIT` constants + labels
- `App\Support\PaymentMethod` — `CASH` / `BKASH` / `NAGAD` / `ROCKET` / `BANK` constants, labels, colors, `isCash()`
- `App\Support\Money` — `format()` + `taka()` helpers (৳ symbol, 2 decimals)
- `App\Services\PaymentService` — `list()` (filters), `listForMember()`, `create()`, `update()`
- `App\Http\Requests\Mess\StorePaymentRequest` + `UpdatePaymentRequest` — admin/super-admin auth, conditional `reference` rule
- `App\Http\Controllers\Mess\PaymentController` — full resource (index/create/store/show/edit/update/destroy) with soft-delete
- `App\Http\Controllers\My\MyPaymentController` — member-scoped history
- `App\View\Components\MethodPill` + `PaymentTypePill` + their Blade views
- Manager views: `index.blade.php`, `create.blade.php`, `edit.blade.php`, `show.blade.php`, `_form.blade.php`, `_list.blade.php`
- Member views: `my._payments.blade.php` partial + Payments tab in `my.blade.php`
- Sidebar nav link in `layouts/app.blade.php` + Payments card on `home.blade.php`
- `Payment` model upgraded with `Auditable` trait + `AuditableContract`
- `PaymentFactory` states: `advanceDeposit()`, `cash()`, `bkash()`, `forMember()`
- Routes: `mess.payments.*` (resource) + `my.payments`
- 4 test files: `PaymentCrudTest`, `PaymentListTest`, `PaymentAuditTest`, `PaymentHistoryTest` — 13 tests total

## Verification

- `composer run test` → 98/98 pass (13 new + 85 prior). No regressions.
- `vendor/bin/pint --test` → clean.

## Decisions honored

- Single form, segmented `Payment type` control (D-01) ✓
- Reference required only for non-cash methods (D-02) ✓
- Manager-only writes (D-06) ✓
- `Mess::activeId()` everywhere `mess_id` is written (taste) ✓
- `__()` everywhere (D-33) ✓
- Decimal money via `Money::taka()` (D-33) ✓
- `DD-MM-YYYY` date format in views (D-34) ✓
- `Auditable` on Payment model (D-30 / AUDIT-01) ✓
- Member history has no filters, no write access (PAY-05) ✓

## Deviations

- Test `test_payments_filter_by_member` rewritten to assert DB row counts instead of HTML presence check, because the dropdown shows all members and an HTML `assertDontSee('Bob')` would always fail.
- Factory `MemberFactory::definition()` changed from `Mess::factory()` to `Mess::activeId() ?? Mess::factory()` so test-created members share the active mess and pass through `MessScope`. Same change applied to `PaymentFactory`.
- `_list.blade.php` makes the show link conditional on `admin`/`super-admin` role so member history doesn't render broken show-route links.
