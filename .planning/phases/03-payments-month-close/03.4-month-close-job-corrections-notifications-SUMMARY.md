## Deviations from Plan

### Resumed after interruption — backend was pre-committed

The prior executor committed the full backend domain layer (8 commits: models, services, job, middleware, controllers, form requests, factories, migrations, route middleware application) but was interrupted before the presentation layer, route registration for the new controllers, notification event wiring, and tests. This session picked up at that boundary and did NOT redo any committed backend work — it verified the backend via tests instead.

### Corrections routes intentionally NOT locked by month.open

The plan's Task 14 lists correction routes inside the `role:admin` group but the plan body (and D-24/CLOSE-12) specifies corrections target closed months. Applying `month.open` to correction routes would make them unreachable. The routes were registered WITHOUT `month.open`, matching the plan's semantic intent. Documented inline in `routes/web.php`.

### Test math adjustment

The plan's example `MonthCloseServiceTest` assertions used `bazar=3000 → meal_rate=100` which conflates meal_rate with bill. The actual math is `meal_rate = total_bazar / total_meals` and `bill = meals × rate`. Tests were written with the correct formulas (e.g. `bazar=60, 60 meals → rate=1 → bill=60 → due=60`) and a dedicated `test_close_numbers_match_bill_preview_service_for_same_inputs` proves the close output equals the preview output byte-for-byte.
