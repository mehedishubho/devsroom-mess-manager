# Quick Task 260723-m1: Mess billing fixes, configurable meal weights, expenses/payments CRUD

**Created:** 2026-07-23
**Status:** Executed

## Goal
Address a batch of user-reported issues across billing, reports, and CRUD:
advance money not reducing the bill, no admin UI for meal weights, ৳0.00 meal
rate across reports/dashboard despite data, no expense edit, missing report
view action, and hidden payment edit/delete.

## Tasks
1. **Meal-rate denominator fix** — `BillPreviewService::compute` summed only
   "denominator-eligible" members (strict joining/leaving bounds), zeroing the
   rate. Sum ALL members' meals (every meal consumed groceries). Fixes Monthly
   Report + Member Statement + Dashboard. Also correct the misleading
   "no bazar recorded yet" label to key off total_bazar.
2. **Configurable meal weights** — `MealType::value()` reads per-mess
   `meal_breakfast/lunch/dinner` settings (already saved by onboarding, never
   read) with cached fallback to 0.5/1/1. Add a Meal-values fieldset to the
   admin Mess settings page + validation + cache invalidation.
3. **Advance offsets the live bill** — preview `due = bill − payments − advance
   used` (capped); month-close consumes the advance for real, carries the rest,
   nets credit vs debt. Add an "Advance applied" column to Bill Preview.
4. **Expenses CRUD** — add show/edit/update/destroy (shared _form partial,
   UpdateExpenseRequest, month.open guards) + a View action on the Expense
   Report + Vendor/Actions columns on the Expenses list.
5. **Payments list actions** — surface Edit + Delete (already in controller)
   in the list table + mobile cards.
6. **Cache invalidation** — confirm new payment/expense invalidates the preview
   (already wired via AppServiceProvider listener) and close the new
   advance-balances adjust gap.

## Decisions (from user)
- Advance money: **reduce the live bill** (auto-deduct from Due; consume at close).
- Total Bazar box shows a real amount → the ৳0.00 bug is purely the denominator.
